# Development Plan (개발 계획)

## 기술 스택

### 프론트엔드 프레임워크
- **React + Vite**
  - 이유: 
    - **가벼움**: Next.js 대비 번들 크기 작음
    - **빠른 개발 서버**: Vite의 HMR로 빠른 개발 경험
    - **단순한 구조**: SPA로 충분, SSR/SSG 불필요
    - **빠른 빌드**: Vite의 esbuild 기반 빌드
    - **운영 비용 0**: 정적 파일만 배포하면 됨
  - Next.js는 이 프로젝트에 과함 (SSR/SSG 불필요, 단일 페이지)

### 언어
- **TypeScript**
  - 이유: 타입 안정성으로 개발 속도 향상 및 유지보수 용이

### 스타일링
- **Tailwind CSS**
  - 이유: 
    - 빠른 개발 속도
    - 반응형 디자인 구현 용이
    - 작은 번들 사이즈
    - 유틸리티 기반으로 간단한 구조

### 지도 API
- **카카오맵 API** (우선 선택)
  - 이유:
    - 무료 티어: 일 300,000건 (월 100명 방문자 기준 충분)
    - 한국 사용자에게 익숙한 지도
    - 장소 검색 API 제공
    - 마커 커스터마이징 용이
  - 대안: 네이버지도 API (무료 티어: 일 100,000건)

### 상태 관리
- **React Context API + useReducer**
  - 이유:
    - 단순한 상태 구조 (위치, 장르, 추천 식당)
    - 추가 라이브러리 불필요
    - 1주 개발 기간에 적합
  - 대안: Zustand (필요시 마이그레이션 가능)

### 폼 관리
- **Native HTML Form + React State**
  - 이유: 
    - 단순한 입력 필드 1개만 있음
    - 추가 라이브러리 불필요
    - 번들 크기 최소화
  - 대안: React Hook Form (필요시 추가)

### HTTP 클라이언트
- **Fetch API (Native)**
  - 이유: 추가 라이브러리 불필요, 브라우저 네이티브 API

### 배포 및 호스팅
- **GitHub Pages** (우선 선택)
  - 이유:
    - **완전 무료**: 무제한 무료 호스팅
    - **운영 비용 $0**: 트래픽 제한 없음
    - **간단한 배포**: 정적 파일만 업로드
    - **자동 배포**: GitHub Actions로 자동화 가능
    - **SSL 인증서**: 자동 제공
  - 대안: 
    - **Netlify**: 무료 티어 (월 100GB 대역폭)
    - **Vercel**: 무료 티어 (월 100GB 대역폭)

### 개발 도구
- **ESLint + Prettier**: 코드 품질 및 포맷팅
- **Git**: 버전 관리

## 주요 기술 결정 사항

### 1. 백엔드 없이 프론트엔드만 사용
- **이유**: 
  - 단순한 기능으로 백엔드 불필요
  - 카카오맵 API를 클라이언트에서 직접 호출
  - 비용 절감 및 구조 단순화
  - 1주 개발 기간에 적합

### 2. 정적 사이트 (Static Site)
- **이유**:
  - 서버 없이 정적 파일만 배포
  - CDN으로 빠른 로딩
  - 운영 비용 $0
  - GitHub Pages 등 무료 호스팅 활용

### 3. 클라이언트 사이드 렌더링 (CSR)
- **이유**:
  - 지도 API 키를 클라이언트에서 사용
  - 실시간 검색 및 상호작용
  - 단순한 구조

### 4. 상태 관리 최소화
- **이유**:
  - 단일 페이지 애플리케이션
  - 복잡한 상태 없음
  - Context API로 충분

## API 사용 계획

### 카카오맵 API
1. **JavaScript SDK**: 지도 렌더링
2. **REST API - 주소 검색**: 주소를 좌표로 변환
3. **REST API - 키워드로 장소 검색**: 식당 검색

### API 호출 제한
- 무료 티어: 일 300,000건
- 예상 사용량: 월 100명 × 평균 5회 검색 = 500건/월
- **충분한 여유**

## 환경 변수

```env
VITE_KAKAO_MAP_API_KEY=your_api_key_here
```

**주의**: Vite는 `VITE_` 접두사를 사용해야 클라이언트에서 접근 가능

## 성능 최적화 전략

1. **코드 스플리팅**: Vite의 자동 코드 스플리팅 활용
2. **번들 최적화**: Vite의 esbuild로 빠른 빌드 및 작은 번들
3. **지도 지연 로딩**: 사용자가 검색할 때만 지도 로드
4. **API 호출 최적화**: 디바운싱 및 캐싱 활용
5. **정적 파일 최적화**: 이미지 압축, CSS/JS minify

## 보안 고려사항

1. **API 키 관리**: 환경 변수로 관리, 클라이언트 노출 최소화
2. **CORS 설정**: 카카오맵 API 도메인 제한 설정
3. **입력 검증**: 사용자 입력 값 검증 및 sanitization

## 확장 가능성

향후 기능 추가 시 고려사항:
- **백엔드 추가**: 사용자 인증, 즐겨찾기 등 (Supabase 무료 티어 활용)
- **데이터베이스**: PostgreSQL (Supabase 무료 티어) 또는 MongoDB Atlas
- **상태 관리**: Zustand 또는 Redux Toolkit으로 마이그레이션
- **테스트**: Vitest (Vite 기반, 빠른 테스트 러너)
- **라우팅**: React Router (필요시)
